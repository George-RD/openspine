## One spine, every signal

Every event — a Telegram message, an email, a timer — travels the same path, in the same order, every time: verify → identify → route → compose → grant → run → gate → audit. Nothing skips a stage. The model reasons; the spine decides.

<CardGrid stagger>
	<Card title="Authority is composed, not assumed" icon="code-branch">
		An agent starts with nothing. Routes, manifests, packs, and policies intersect — deterministically, deny-by-default — into one short-lived task grant. That grant is the only authority the agent ever holds.
	</Card>
	<Card title="One gate on every effect" icon="forward-slash">
		Before any connector touches the world, the action passes a single gate: allow, deny, or ask you first. Every decision is appended to a hash-chained audit log that references encrypted artifacts, never plaintext.
	</Card>
	<Card title="Claims you can falsify" icon="approve-check">
		Every documented security claim maps to a named test, and the build fails if a claimed test disappears. If we say the agent can't send email, there's a test you can run that proves it.
	</Card>
</CardGrid>

## Trust grows at the speed of your approvals

Agents on OpenSpine can propose new rules, routes, and capabilities — and can never activate them. What you approve is digest-bound: byte-for-byte what runs. No tool store, no silent capability creep, no agent deciding it needs more access.
